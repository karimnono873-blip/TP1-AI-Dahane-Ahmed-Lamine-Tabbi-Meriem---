1. Problématique & Notions Théoriques

Dans un contexte commercial de plus en plus compétitif, une entreprise souhaite classer automatiquement ses clients afin d'adapter et de personnaliser sa stratégie marketing. Comprendre qui sont les clients permet de cibler les campagnes publicitaires, de fidéliser la clientèle et d'optimiser les ventes. Le jeu de données utilisé pour ce TP est le Mall Customer Dataset, qui contient des informations démographiques et comportementales (âge, sexe, score de dépense, revenu annuel).

L'objectif de notre travail est de segmenter les clients selon leurs revenus en utilisant un algorithme d'apprentissage supervisé puissant : le Random Forest (Forêt Aléatoire). En prédisant la catégorie de revenu d'un client en fonction de son comportement, l'entreprise peut anticiper ses besoins.

Comment fonctionne l'algorithme Random Forest ?

Le Random Forest est un algorithme d'Ensemble Learning (apprentissage ensembliste). Plutôt que de s'appuyer sur un seul modèle prédictif qui pourrait se tromper ou manquer de nuances, il combine les prédictions de multiples modèles plus simples pour obtenir un résultat final plus robuste et précis. Son fonctionnement repose sur la technique du Bagging (Bootstrap Aggregating) :

  • Création d'arbres multiples : L'algorithme génère une multitude (une "forêt") d'arbres de décision indépendants pendant la phase d'apprentissage.
  • Le Bootstrap : Chaque arbre est entraîné sur un sous-échantillon aléatoire des données d'entraînement, tiré avec remise. Ainsi, chaque arbre a une vision légèrement différente du problème.
  • Sélection aléatoire des variables (Features) : À chaque étape de la division d'un nœud dans un arbre, seul un sous-ensemble aléatoire des variables est pris en compte. Cela force les arbres à être diversifiés et empêche une variable dominante de masquer l'utilité des autres.
  • Agrégation (Vote majoritaire) : Lors de la prédiction, chaque arbre "vote" pour une classe. La classe qui récolte le plus grand nombre de votes devient la prédiction finale du modèle.

Avantages et Inconvénients

Avantages

  • Haute précision : Grâce à l'agrégation, il corrige naturellement les erreurs des arbres individuels.
  • Robuste au surapprentissage (Overfitting) : La diversité des arbres empêche le modèle d'apprendre par cœur les données d'entraînement.
  • Importance des variables : Il permet d'identifier facilement quelles données (ex: l'âge ou le score de dépense) influencent le plus la décision.

Inconvénients

  • Effet "Boîte Noire" : Contrairement à un arbre de décision unique, il est difficile de visualiser et d'interpréter le chemin de décision exact d'une forêt entière.
  • Coût en calcul : L'entraînement et la prédiction prennent plus de temps et de mémoire, surtout avec un grand nombre d'arbres.

2. Préparation des Données et Évaluation

La Labellisation et le Nettoyage

Le jeu de données original n'est pas labellisé pour la classification (il s'agit à l'origine d'un problème non supervisé). Nous avons donc créé notre propre cible (Target) en segmentant la colonne Annual Income (k$) en 3 classes : Faible, Moyen, et Élevé.

Pourquoi supprimer l'ID et la colonne cible des Features ?

Avant d'entraîner notre modèle IA, nous devons séparer les variables explicatives (X) de la variable à prédire (y). Il est crucial de supprimer la colonne Cible de X. Si on la laissait, le modèle "tricherait" en apprenant la réponse directement au lieu d'en déduire la logique.
De plus, nous devons supprimer la colonne CustomerID. L'ID n'est qu'un numéro de série abstrait. Le fournir au modèle risquerait de créer du bruit statistique : l'algorithme pourrait tenter de trouver des règles mathématiques basées sur ces identifiants qui n'ont aucune réalité comportementale.

Comprendre les Métriques d'Évaluation

Pour mesurer les performances de notre Random Forest, nous utilisons des outils statistiques précis :

  • La Matrice de Confusion : C'est un tableau qui croise les prédictions de notre IA avec la réalité. Elle permet de visualiser exactement où le modèle se trompe. Elle met en évidence les Vrais Positifs (bien prédit), Vrais Négatifs, Faux Positifs (le modèle a prédit positif à tort) et Faux Négatifs (le modèle a raté un cas positif).
  • La Précision (Precision) : Elle répond à la question : "Parmi tous les clients que le modèle a classés comme 'Revenus Élevés', combien le sont réellement ?". Une haute précision signifie que le modèle fait très peu de fausses alertes.
  • Le Rapport de Classification : Il compile plusieurs métriques pour chaque classe (Faible, Moyen, Élevé). Outre la Précision, il inclut le Rappel (Recall), qui indique la capacité du modèle à trouver tous les cas d'une classe donnée, et le F1-Score, qui est la moyenne harmonique entre la précision et le rappel (idéal pour les ensembles de données déséquilibrés).

3. Implémentation Python (Solution Complète)

Voici le code complet, détaillé et commenté, permettant de résoudre cette problématique en Python :


import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report, confusion_matrix
from sklearn.preprocessing import LabelEncoder

# 1. Chargement des données
df = pd.read_csv('Mall_Customers.csv')

# 2. Labellisation (Création des 3 classes de revenus)
# Catégorisation : < 40k (Faible), 40k-70k (Moyen), > 70k (Élevé)
bins = [0, 40, 70, 150]
labels_revenu = ['Faible', 'Moyen', 'Élevé']
df['Income_Class'] = pd.cut(df['Annual Income (k$)'], bins=bins, labels=labels_revenu)

# 3. Encodage des variables catégorielles (Gender)
# Les algorithmes mathématiques requièrent des nombres
le = LabelEncoder()
df['Gender'] = le.fit_transform(df['Gender']) # Male=1, Female=0

# 4. Séparation Features (X) et Target (y)
# Suppression de l'ID (bruit) et de la cible (pour éviter la triche)
X = df.drop(['CustomerID', 'Annual Income (k$)', 'Income_Class'], axis=1)
y = df['Income_Class']

# 5. Split Train/Test (70% apprentissage, 30% test)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 6. Initialisation et Entraînement du modèle Random Forest
# Création d'une forêt de 100 arbres
rf_model = RandomForestClassifier(n_estimators=100, random_state=42)
rf_model.fit(X_train, y_train)

# 7. Prédictions sur l'ensemble de test
y_pred = rf_model.predict(X_test)

# 8. Évaluation du modèle
print("=== Matrice de Confusion ===")
print(confusion_matrix(y_test, y_pred))

print("\n=== Rapport de Classification ===")
print(classification_report(y_test, y_pred))
